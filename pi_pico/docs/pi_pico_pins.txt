SPECIAL USE PINS  ====================================

GPIO 23, 24, 25, 29: These are used internally on the board (SMPS Power save, VBUS Sense, Onboard LED, VSYS measurement).
    GPIO 23 SMPS (switched mode power supply): Power save
    GPIO 24 VBUS Sense
    GPIO 25 Onboard LED
    GPIO 29 VSYS voltage

    GPIO 0, 1:           Default UART0 TX/RX.  Secondary: 
    GPIO 4, 5:           Default I2C0 SDA/SCL.
    GPIO 16, 17, 18, 19: Default SPI0.
    GPIO 26, 27, 28:     ADC (Analog-to-Digital)


These pins are best for basic digital on/off switching or simple input because they lack default peripheral mappings (like the first I2C/UART): 
BEST PINS for simple i/o and switching
    GPIO 10, GPIO 11, GPIO 12, GPIO 13, GPIO 14, GPIO 15, GPIO 19, GPIO 20, GPIO 21, GPIO 22.
    GP14/15 are used for UART1 default, but are freely available if not using UART1. 


ADC PINs  ===========================================

Potentiometer: use 10K on ADC pins:
  * Optimal Input Impedance: The RP2040 chip's ADC is optimized for sources 
      with an output impedance of 10kΩ or less. A 10kΩ potentiometer provides 
      a good balance between accuracy and current draw.
  * Low Power Consumption: keeps current draw low (approx 0.33mA at 3.3V) 
      while allowing the Analog-to-Digital Converter's sampling capacitor 
      to charge quickly enough for accurate readings.
  * Standard Component: widely available, linear, and work perfectly with 
     the 3.3V logic used by the Pico. Use the B / LINEAR pots, not the
     A / AUDIO pots (logarithmic).
  * Lower ohms (1K, 5K) may be a bit more accurate but draw more current.
  * Higher (100K) do not allow the ADC's capacitor to charge fully,
      so values are erratic.
  * Connect outer pins of pot to +3.3v and GND. Center/wiper to ADC pin.

- ADC VREF pin  (physical pin 35)

 * Default Behavior: Without an external connection, the Pico uses the
   3.3v supply (filtered through 200 ohm & 1uF as the reference. 
   This can be noisy and vary depending on the power source.
 * Improving Accuracy: For better performance, connect an external voltage 
   reference (e.g. 3.0V ref) to stabilize ref voltage, gives more accuracy.
 * Using 3.3v is not recommended but can work.
 * Analog Ground: Use the separate AGND pin (Pin 33) as the ground 
   for your analog sensors to reduce digital noise. 

- STABLIZING THE POWER SUPPLY

If you do not supply 3.0v to ADC VREF, Pico uses the 3.3v supply (physical 36).
The voltage is not stable because the power supply (PS) uses 2 modes.
GPIO23 controls the RT6150 PS (Power Save) pin. When PS is low (the default
on Pico) the regulator is in Pulse Frequency Modulation mode.
At light loads, this saves considerable power by only turning on the 
switching MOSFETs occasionally to keep the output capacitor topped up. 
Setting PS high forces the regulator into Pulse Width Modulation (PWM) mode
constantly.
PWM mode forces the SMPS to switch continuously, which reduces the 
output ripple considerably at light loads but at the expense of much 
worse efficiency. The PFM mode can work well but the flipping between
PFM  and PWM modes causes noise in the 3.3v supply.
Under heavy load the switcher will be in PWM mode always, regardless of
the PS pin state.


POWER PINS  =====================================

POWER LIMITS
  50ma max combined GPIO
  3.3v pin can support 300mA (in addition to the 50mA on GPIO)
  A single GPIO has 12 to 16mA max.
  VBUS 5v  750mA max.

These are not GPIO numbered - they use 'raw' hardware pin numbers (1-40)

HW   NAME      FUNCTION
PIN
---- -------- -----------------------------------------------------------
40   VBUS     5v directly from USB  750mA max.
39   VSYS     1.8 - 5.5v power to Pico  (See note below)
38   GND
37   3V3_EN   Connects to the board SMPS enable pin. It is a hardware power 
                on/off (hardware reset). Set it to LOW to reset.
36   3V3_OUT  Supply power to low-power accessories. Max 300mA
                This max excludes the power used by the Pico / Pico-2 chip; 
                so all 300mA is available for accessories.
35   ADC_REF  ADC pin power supply voltage;
33   AGND     GND reference for the ADC pins;
30   RUN      The enable pin; set to LOW to reset the microcontroller chip
                RP2040 (software reset).

GND pins: 3  8  13  18  23  28  33 (33 is ADC GND)   38


=== VBUS and VSYS  ========================================

VBUS (pin 40) is connected directly to USB.

VSYS (pin 39) used to power the Pico if USB is not used.
   If power is supplied to VSYS, no voltage appears on VBUS
   (VBUS is on the reverse side of a Shottky diode).

The diode causes a small drop (0.3 - 0.4v) from VBUS to VSYS.
