 VBUS  pin 40.  provides power direct from USB pin D1.
                Monitor using GPIO 24 (no physical pin).

 VSYS  pin 39. Main 'power in' feed.  1.8 - 5.5 V input. 
               When USB provides power, VSYS is the input volts from VBUS
               minus Schottky diode drop (usu. 5v - (0.15 to 0.45) or 
               4.5-4.9v.
               This loss can be eliminated by shorting VSYS - VBUS but ONLY IF
               the power source is EXCLUSIVELY the USB / VBUS.
               If an external voltage is applied to VSYS, it should be through
               a Schottky diode to protect against backfeed of the USB.
               This allows having voltage applied to both VSYS and VBUS -
               the Pico has a diode to prevent backfeeding from VSYS,
               and the external supply's Schottky protects against
               backfeeding from VBUS into VSYS. Allows for redundant power.
               A better way is to use a MOSFET ahead of VSYS.
               See raspberrypi.com Pi Pico data sheet for details.
               Also shows how to power using a battery and its charger.
 3v3(OUT) pin 36. Can provide 300mA max (extra. in addition to powering the Pico)

 --- QWIIC PINOUT  -----

Qwiic Pinout & Wire Colors
Pin Wire Color   Signal Name  Description
 1 Black  GND (Ground)  Reference ground
 2 Red   3.3V  Power in (regulated 3.3V)
 3 Blue   SDA  (Data) I2C Serial Data
 4 Yellow SCL  (Clock) I2C Serial Clock
Orientation
When looking directly into a female Qwiic connector on a board 
(or facing the locking tab up), the pins typically map from left to right 
as: Black (GND)   Red (3.3V)   Blue (SDA)   Yellow (SCL)

 --- I2C ---------------------

1. The Cable's Capacitance
 Standard limit: Total bus capacitance (including the devices and the cable) 
must stay under 400 pF.
Rule of thumb: Capacitance varies by wire type (standard shielded cables 
usually range from 100 to 240 pF per meter. Therefore, a 2 to 3-meter 
cable typically hits this limit.
Best practice: Use unshielded twisted pair (UTP) or low-capacitance wire, 
and always keep it as short as possible.

2. Clock Speed
 At standard I2C clock speeds (like \(100 \text{ kHz}\)), you may only get about 1 to 2 meters.Dropping the clock speed to \(10 \text{ kHz}\) or \(5 \text{ kHz}\) reduces the signal's rise-time requirements, letting you reach up to 10 meters.

3. Pull-Up Resistors
 You can sometimes improve signal reliability over longer cables (up to 2–3 meters) by using physically stronger pull-up resistors (e.g., \(2.4 \text{ k}\Omega\) instead of \(10 \text{ k}\Omega\)). This helps overcome the inherent capacitance of the wire

How to achieve longer distances (10+ meters)

If your application requires distances greater than 10 meters, 
standard I2C is not suitable without hardware additions. 
You can bypass length limits by using:I2C Bus Extenders / Active Terminators: 
Use specialized ICs like the Adafruit LTC4311 I2C Extender to actively drive 
the bus over longer distances.
Differential Extenders: 
Use specialized chips (like the NXP P82B96) that convert the single-ended 
I2C signal into a differential signal, allowing lengths of up to 
hundreds of meters.

Alternative Protocols: If you need to span long distances (10+ meters), 
it is usually more reliable to use a protocol designed for it, 
such as CAN bus or RS-485.

=== ADC VREF  and ADC GPIO pins  ==================================

The Pico 2 has improved accuracy vs Pico for measuring ADC analog.
It may be good enough to not bother to provide ADC_VREF.
The Pico 2 then defaults to using the 3.3v supply.

Higher accuracy is possible by providing a reference voltage to
ADC_VREF pin.

NOISE: add 0.1uV cap on the ADC Pin to reduce noise.

The ADC_VREF pin max allowed is 3.0 v.
The output from many sensors is higher than that.
Ex: DFRobot sensor needs at least 3.3v to operate, so it will put
3.3v on the ADC pin (ex GPIO26). So this voltage must be reduced
to 3 volts or less. 
Using a resistor divider:
 Sensor Signal
  0V to 3.3V ---- R1: 1kΩ  ----+---- GPIO 26 (0V to 3.0V)
                               |
                            R2: 10kΩ 
                               |
                              GND

                         R_2                10k
   V_out  = V_in *  ------------  =  3.3v * --------  = 3.3v - 0.909 = 3v
                      R_1 + R_2            1k + 10k

ADC_VREF 
The best accuracy for the ADC is obtained if you use a stable reference
voltage supplied to the ADC_VREF pin (physical 35).

--- OUTDOOR WIRING  -----------------------------

Use Direct Burial or UV-Resistant CAT-5 or CAT-6 cable.
Marine-grade adhesive-lined heat shrink tubing or silicone-filled waterproof
junction box to protect connections esp. at the sensor.
Could use PVC conduit to run the wire.

Use CAT wire like this: how to use the twisted pairs in the cable:
  pair 1: Power & Gnd.  Use solid color wire for VCC 3.3v and 
          striped-color pair-partner for GND.
  pair 2: Signal & Gnd  Use solid color for signal, striped for GND.
Connect the GNDs at the Pico end - connect to Pico GND pin.
  
Could use 5v to the sensor instead of 3.3v. Need a resistor divider
as described above.

=== RELAY MODULE BOARD  ==================================

See
  ~/git/microcontrollers/components/relay_module_3.3v/relay_board.txt
  ~/git/microcontrollers/components/relay_module_3.3v/relay_module_3.3-5v.pdf

Use Pico GPIO pin (in output mode) to provide power to IN terminal.
Connect Pico GND to the GND on the relay board.
To power the board components (transistor, relay), use VCC.

Power: 
  Input for IN terminal (ie input to optoisolator): at 3.3v uses 2.4 mA.
  Transistor+Coil at 3.3v uses 110 mA.
  Relay points ('output') can handle 10A at either 250VAC or 30VDC.

Jumpers:
Connect J1, J2 to power the board from Pico. Connect 3V3 to board's
VCC terminal. Connect a GPIO pin to IN terminal.
Connect a Pico GND to GND.

If you remove J2, it isolates the optoisolator input from the rest of
the board. Then provide signal from GPIO pin to IN terminal and Pico
GND to the GND terminal. To power the board (ie the components incl.
coil, transistor, output side of optoisolator), use VCC (or J1B if you
remove J1) and use J2B to supply GND. This allows complete separation
of the input and relay circuits.
